# 上传项目到 GitHub 的执行计划

本计划旨在将当前工作目录中的所有更改（包括 AI 模块、后端 Java 代码以及前端 UI 的更新）安全地推送到远程 GitHub 仓库 `relifel/CryptoRate`。

## 任务目标
1. 暂存所有已修改和未跟踪的文件。
2. 编写符合规范的中文提交信息。
3. 将本地代码推送到 GitHub 远程仓库的 `main` 分支。

## 执行步骤

| 步骤 | 操作 | 说明 |
| :--- | :--- | :--- |
| 1 | **代码暂存 (Stage)** | 使用 `git add .` 将所有变更添加到暂存区。 |
| 2 | **代码提交 (Commit)** | 编写中文提交信息，概括本次更新的主要内容。 |
| 3 | **代码推送 (Push)** | 执行 `git push origin main` 将本地代码同步到 GitHub。 |
